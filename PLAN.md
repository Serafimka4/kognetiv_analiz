# ПО для когнитивного анализа рыночных тенденций (MOEX)

> Дипломный проект. Подход: ML / NLP (cognitive computing). Стек: Python.
> Рынок: российские акции Московской биржи (MOEX).

## 1. Концепция

Система собирает рыночные данные (котировки MOEX) и русскоязычные тексты
(новости, аналитика), извлекает из текстов «когнитивные» признаки — тональность,
темы, события — и связывает их с поведением рынка для **анализа и прогноза
тенденций** с объяснением факторов влияния.

**Ключевая гипотеза диплома:** учёт текстовой аналитики (настроения новостей)
повышает точность прогноза направления тренда по сравнению с моделью только
на ценовых данных.

## 2. Архитектура

```
Сбор данных → Предобработка → NLP-анализ → Формирование признаков
   (MOEX,        (очистка,      (тональность,    (цена + тех.индикаторы
    новости)     лемматизация)   темы, NER)       + индекс настроений)
                                                          ↓
                                                    ML-прогноз тренда
                                                          ↓
                              Когнитивный слой (связь настроения↔цена,
                                       объяснимость SHAP, сценарии)
                                                          ↓
                                          Визуализация / Dashboard
```

## 3. Источники данных (под MOEX)

| Данные | Источник | Инструмент |
|---|---|---|
| Котировки акций, индексы (IMOEX) | ISS MOEX API | `apimoex`, `requests` |
| Дневные/внутридневные ряды | ISS MOEX | `apimoex` |
| Новости (РБК, Интерфакс, Финам, Прайм) | RSS-ленты | `feedparser` |
| Доп. новостной поток | парсинг сайтов | `requests` + `beautifulsoup4` |

Тикеры для прототипа: ликвидные «голубые фишки» — SBER, GAZP, LKOH, GMKN, YNDX.

## 4. Технологический стек

- **Данные:** `apimoex`, `pandas`, `feedparser`, `beautifulsoup4`
- **Хранение:** `SQLite` (достаточно для диплома)
- **Предобработка рус. текста:** `razdel`, `pymorphy3`, `natasha`
- **NLP-модели:** `transformers` (RuBERT для тональности), `gensim` (LDA для тем)
- **Признаки/индикаторы:** `pandas`, `ta`
- **ML:** `scikit-learn`, `xgboost`
- **Объяснимость:** `shap`
- **UI:** `Streamlit`
- **Графики:** `plotly`, `matplotlib`

## 5. Методы (научное ядро)

1. **Тональность новостей** — RuBERT (дообученная на фин. текстах модель,
   напр. `blanchefort/rubert-base-cased-sentiment`) → числовой индекс настроения
   по дням.
2. **Тематическое моделирование (LDA)** — выделение драйверов рынка
   («санкции», «дивиденды», «ставка ЦБ», «отчётность»).
3. **NER (natasha)** — привязка новостей к конкретным компаниям/тикерам.
4. **Формирование признаков** — лаги цены, доходности, тех. индикаторы
   (RSI, MACD, скользящие средние) + агрегированный индекс настроений + частоты тем.
5. **Прогноз тренда** — классификация направления (вверх / вниз / боковик),
   модель XGBoost; baseline — логистическая регрессия и «наивный» прогноз.
6. **Объяснимость (SHAP)** — вклад каждого фактора в прогноз = когнитивная
   интерпретация.
7. **Оценка** — accuracy / F1 / precision-recall, walk-forward backtest;
   главный эксперимент: сравнение **«цена + тексты» vs «только цена»**.

## 6. Этапы реализации

- [x] **Этап 1.** Сбор данных: индекс IMOEX + акции MOEX (`apimoex`) + архив новостей
      Lenta.ru (рубрика «Экономика»), запись в SQLite. ✅
      Запуск: `python collect.py`. БД: `data/market.db`.
      Целевой объект — **индекс IMOEX** (макро-новости Lenta хорошо ложатся на индекс).
      **Окно: 2015-01-01..2018-12-31** (рубрика «Экономика» в архиве Lenta заканчивается 2018 г.).
      Итог: IMOEX — 1008 дней, акции — 4032 строки, новостей — 14 767.
      Архив: `data/raw/lenta-ru-news.csv.bz2` (800K статей, 1999–2019).
      RSS-сборщик (`src/collectors/news.py`) оставлен для живого демо на свежих данных.
- [x] **Этап 2.** Предобработка русского текста: очистка (`clean_text`) + лемматизация
      (`razdel` + `pymorphy3`, стоп-слова) → таблица `news_clean`. ✅
      Запуск: `python preprocess.py`. Два представления на новость:
      `clean_text` (для RuBERT) и `lemmas` (для LDA). Обработано 14 767 новостей.
- [x] **Этап 3.** NLP: тональность (RuBERT `blanchefort`) → `news_sentiment`;
      темы (LDA, 8 тем) → `news_topic`. ✅ Запуск: `python -m src.nlp.sentiment` и
      `python -m src.nlp.topics`.
      Темы интерпретируемы (нефть, валюта, санкции, банки, бюджет, Украина).
      ⚠️ НАХОДКА: дневной индекс настроений по архиву Lenta слабо коррелирует с
      доходностью IMOEX (|r|<0.09 даже по рыночным темам и со сглаживанием), с
      волатильностью чуть лучше, но тоже слабо. Причины: негативное смещение модели
      + общий (не рыночно-специфичный) характер новостей Lenta «Экономика».
      Требуется стратегическое решение (см. ниже).
- [x] **Этап 4.** Датасет признаков: 11 ценовых (px_*: лаги, волатильность, RSI, MACD,
      SMA, моментум) + 13 текстовых (tx_*: тональность, объём, темы) → `data/dataset.csv`.
      975 дней, цель y = знак доходности IMOEX на след. день. ✅
      Запуск: `python -m src.features.build_dataset`.
- [x] **Этап 5.** XGBoost + walk-forward (TimeSeriesSplit): сравнение
      baseline / price / text / price+text. ✅ `python -m src.models.train`.
      Результат: price=0.480, text=0.506, **price+text=0.507** (baseline 0.489),
      Δ=+2.7 п.п. в пользу текста, но ROC-AUC≈0.51 (близко к случайной).
      Честный вывод: текст слегка улучшает прогноз, но абсолютная сила низкая —
      согласуется с эффективностью рынка и общим характером новостей Lenta.
- [x] **Этап 6.** Когнитивный слой: SHAP-важность признаков модели price+text. ✅
      `python -m src.cognitive.explain` → `data/shap_importance.csv`,
      `data/figures/shap_importance.png`. Вклад групп: цена 52% / текст 48%.
      Топ текстовых факторов — отраслевые настроения (банки, валюта, санкции, нефть),
      т.е. тематическая тональность информативнее общей.
- [x] **Этап 7.** Дашборд на Streamlit: 6 вкладок — Обзор (IMOEX + индекс настроений),
      Настроения, Темы-драйверы, Прогноз и факторы (SHAP), Выводы, 🔴 Сейчас (live). ✅
      Запуск: `streamlit run src/dashboard/app.py`.
- [x] **Live-мониторинг.** `src/live/monitor.py`: свежие RSS-новости + текущие
      котировки IMOEX → тональность (RuBERT) + тема (сохранённая LDA) →
      таблицы `live_index`/`live_news` → вкладка «Сейчас». ✅
      Запуск: `python -m src.live.monitor` (накапливает данные при повторных запусках).
- [ ] **Этап 8.** Эксперименты, сравнение моделей, оформление пояснительной записки.
